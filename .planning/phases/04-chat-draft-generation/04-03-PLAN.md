**ChatMessage.svelte** — Renders a single chat message with draft block support:
- Props: `message` (UIMessage-shaped: { id, role, parts, createdAt? }), `draftEdits` (array of edits for this message, optional)
- Layout: Message bubble with alignment — user messages right-aligned (bg-gray-900 text-white), assistant messages left-aligned (bg-white border), system messages centered (muted, italic)
- For assistant messages, parse text parts for `<draft>...</draft>` markers:
  - Create a `parseDraftBlocks(text)` function that splits text into segments of type 'text' or 'draft'
  - Render 'text' segments as regular text (with basic markdown-like line break handling)
  - Render 'draft' segments as `<DraftBlock>` components
  - IMPORTANT: Only parse draft markers when the text part's state is NOT 'streaming' (during streaming, render raw text to avoid partial-tag glitches). Check `part.state !== 'streaming'` or fall back to rendering raw text.
- For user messages, render the text parts directly
- Iterate over `message.parts` — for each part where `part.type === 'text'`, render the text (with draft parsing for assistant)
- Small timestamp at the bottom of each message if createdAt is available

**ChatInterface.svelte** — Main chat container orchestrating the Chat class:
- Props: `postId` (string), `initialMessages` (UIMessage[]), `personas` (array of {id, name}), `activePersonaId` (string | null), `draftEdits` (array of all draft edits for this post)
- Initialize the AI SDK Chat class:
  ```
  import { Chat } from '@ai-sdk/svelte';
  import { DefaultChatTransport } from 'ai';
  ```
  Create a Chat instance with:
  - `id`: postId
  - `messages`: initialMessages (pre-loaded from DB)
  - `transport`: new DefaultChatTransport with `api: /queue/${postId}/chat` and dynamic `body: () => ({ personaId: currentPersonaId })`
  - `onError`: set an error state for display
- State management (Svelte 5 runes):
  - `currentPersonaId` ($state): initialized from activePersonaId prop
  - `inputText` ($state): chat input text
  - `error` ($state): error message from chat errors
  - `isStreaming` ($derived): `chat.status === 'streaming'`
- UI layout (top to bottom):
  1. Message list in a scrollable container (flex-1, overflow-y-auto). Use `{#each chat.messages as message (message.id)}` — DO NOT destructure chat.messages. Render each with `<ChatMessage>`.
  2. Auto-scroll to bottom when new messages arrive (use $effect watching chat.messages.length to scroll container)
  3. PersonaSelector below the message list
  4. Input area: textarea (auto-expanding or fixed 2 rows) + send button. On submit (Enter key or button click), call `chat.sendMessage({ text: inputText })` and clear inputText.
  5. Show a typing indicator when isStreaming (e.g., "AI is typing..." with subtle animation)
  6. Show error message if error state is set (dismissible)
- Disable send button when inputText is empty or isStreaming
- Handle Enter to send (Shift+Enter for newline)
