  **Install AI SDK dependencies:**
  ```bash
  bun add ai @ai-sdk/svelte @ai-sdk/anthropic
  ```

  **Update `.env.example`** — Add `ANTHROPIC_API_KEY=your-anthropic-api-key-here` below the existing entries.

  **Update `src/app.d.ts`** — Add `ANTHROPIC_API_KEY: string` to the `App.Platform` env interface (alongside existing `DATABASE_URL`).

  **Create `src/lib/server/services/voice.ts`** — Voice profile service factory (same pattern as persona service):

  ```typescript
  export function createVoiceService(db: Awaited<ReturnType<typeof getDb>>) { ... }
  ```

  Methods:
  - `saveSamples(personaId, content, platform)` — INSERT into writingSamples table. Calculate wordCount from content (split by whitespace). Return the sample record.
  - `getSamples(personaId)` — SELECT all writing samples for persona, ordered by createdAt DESC.
  - `saveVersion(personaId, extractedProfile, sampleStats)` — Calculate next version number (MAX version for persona + 1, default 1). INSERT into voiceProfileVersions with extractedProfile JSONB, sampleCount, samplePlatforms, totalWordCount. UPDATE persona's `activeVoiceVersionId` to the new version's id. Return the version record.
  - `listVersions(personaId)` — SELECT all versions for persona, ordered by version DESC. Return id, version number, createdAt, sampleCount, totalWordCount.
  - `getVersion(personaId, versionId)` — SELECT single version with ownership check via persona.
  - `getActiveVersion(personaId)` — SELECT the version matching persona's `activeVoiceVersionId`. Return full version data including extractedProfile and manualEdits.
  - `setActiveVersion(personaId, versionId)` — UPDATE persona's `activeVoiceVersionId`.
  - `updateManualEdits(versionId, manualEdits)` — UPDATE the manualEdits JSONB on a specific version. This stores user overrides separately from the AI extraction.

  **Create `src/routes/(app)/personas/[id]/voice/+server.ts`** — Streaming API endpoint for voice extraction:

  - POST handler: Check auth (`await locals.auth()`), return 401 if no session. Parse request JSON for `{ samples: string, platform: string }`. Validate: samples must be at least 100 characters, platform must be one of the allowed values.
  - Get ANTHROPIC_API_KEY via `platform?.env?.ANTHROPIC_API_KEY ?? env.ANTHROPIC_API_KEY` (import env from `$env/dynamic/private`). Error if missing.
  - Get DB via `getDb(databaseUrl)`, create voice service.
  - Save writing samples to DB first (call `voice.saveSamples()`).
  - Get all existing samples for persona (for cumulative extraction).
  - Use Vercel AI SDK `streamText` with `Output.object({ schema: voiceProfileSchema })`:
    - Model: `anthropic('claude-sonnet-4-20250514')` (create provider per-request with explicit apiKey, NOT module-scope).
    - System prompt: Expert writing style analyst. Analyze provided writing samples and extract a detailed voice profile. Be specific — cite actual patterns, phrases, and tendencies. The profile should capture what makes this writer's voice unique.
    - User prompt: Include ALL writing samples for this persona (cumulative), tagged with their platform source.
  - Use `onFinish` callback on `streamText` to save the completed extraction as a new version via `voice.saveVersion()`. Parse the final output from the callback's `experimental_output` field.
  - Return `result.toTextStreamResponse()` for streaming to client.

  **CRITICAL patterns:**
  - Create `createAnthropic({ apiKey })` per-request, NOT at module scope (Workers reuse isolates)
  - Access env vars same way as Phase 1 DB URL pattern
  - Auth check is REQUIRED in `+server.ts` — layout auth guards don't apply to API routes (Research Pitfall 4)
  - Use `streamText` + `Output.object()`, NOT deprecated `streamObject` (Research: AI SDK v6)
  </action>

  - Manual test: `curl -X POST http://localhost:5173/personas/[test-id]/voice -H "Content-Type: application/json" -d '{"samples":"test","platform":"twitter"}' -H "Cookie: ..."` returns streaming response (or 401 without auth)
  </verify>

  AI SDK installed and configured. Voice service handles sample storage and version management. Streaming API endpoint extracts voice profile from writing samples using Claude and saves the result as a versioned profile.

  **Create `src/lib/components/persona/VoiceExtractor.svelte`** — Writing sample input + streaming extraction:

  Props: `personaId: string`, event callback `onExtracted: (profile) => void`
  State: `samples` (string, bound to textarea), `platform` (string, bound to dropdown), `extracting` (boolean), `partialProfile` (Partial<VoiceProfile> | null), `error` (string | null), `wordCount` (derived from samples)

  UI layout:
  - Textarea: large text area (min-h-48) with placeholder "Paste your writing samples here — tweets, blog posts, emails, anything that represents how you write..." Bind to `samples` state.
  - Word counter: Show live word count below textarea. Display guidance: "500+ words recommended for accurate voice extraction". Show warning styling if < 100 words. Use Tailwind text-sm text-gray-500.
  - Platform dropdown: Select with options: Twitter, LinkedIn, Blog, Reddit, Email, Other. Default to "twitter".
  - Extract button: "Extract Voice Profile" — disabled when extracting or samples < 100 words. Show spinner icon when extracting.

  Streaming extraction flow:
  - On button click, `fetch POST` to `/personas/${personaId}/voice` with `{ samples, platform }`.
  - Read response body as ReadableStream. Decode chunks with TextDecoder. Accumulate text.
  - The AI SDK `toTextStreamResponse()` with `Output.object()` sends text chunks. Parse accumulated text — attempt `JSON.parse()` on each accumulation. On success, update `partialProfile`. On parse error, continue (partial JSON, wait for more data).
  - During streaming: show the VoiceProfileDisplay component with the partial profile, traits appearing progressively as they're extracted. Show a pulsing "Analyzing your writing..." indicator.
  - On stream completion: set `extracting = false`, call `onExtracted` callback. Clear the textarea.
  - On error: show error message with retry button.

  **Create `src/lib/components/persona/VoiceProfileDisplay.svelte`** — Voice profile rendering:

  Props: `profile: VoiceProfile | Partial<VoiceProfile>`, `manualEdits: Record<string, unknown> | null`, `editable: boolean` (default false), `streaming: boolean` (default false)

  UI layout (per user decision: "natural language summary at top, structured traits below"):
  - **Summary section**: Natural language summary paragraph at top. If streaming, show skeleton/pulse animation until summary appears.
  - **Structured traits** (below summary, in a grid or card layout):

    - **Vocabulary**: Level badge, "Favorites" list (as small tags/pills), "Avoids" list (as crossed-out or red-tinted tags), "Jargon" list (as tags)
    - **Sentence Structure**: Average length badge, complexity badge, patterns as bullet list
    - **Quirks**: List of distinctive habits as bullet points
    - **Rhetoric**: Persuasion style, opening style, closing style as descriptive text; transitions as tag list
  - If `editable` is true: each trait value is clickable/editable. Clicking a trait opens inline edit (input or select). Changes are collected as `manualEdits` and dispatched via callback `onEdit(edits)`. Display applied manual edits with a subtle "edited" indicator and show the original value on hover.
  - If `streaming` is true: show skeleton placeholders for sections that haven't arrived yet. Sections animate in as data appears.
  - If `manualEdits` exist: merge with extractedProfile for display, showing edited values with a small "edited" badge.
