**Schema change** — Add `platform` column to voiceProfileVersions:

- `platform`: varchar(50), nullable — null means "default" (applies to all platforms without a specific override)
- Add composite index on (personaId, platform) for efficient lookup
- The existing `activeVoiceVersionId` on personas remains the DEFAULT voice. Platform overrides are looked up separately.

Run `bunx drizzle-kit push` to apply.

**Extend voice service** with platform-aware methods:

```typescript
async getActiveVersionForPlatform(personaId: string, platform: string | null): Promise<VoiceProfileVersion | null>
// 1. First, look for a platform-specific version: query voiceProfileVersions where personaId matches AND platform matches, ordered by version desc, limit 1
// 2. If found, return it
// 3. If not found, fall back to getActiveVersion(personaId) — the default voice
// This implements the "platform override with default fallback" pattern

async listPlatformOverrides(personaId: string): Promise<Array<{ platform: string; version: VoiceProfileVersion }>>
// Get all platform-specific versions (where platform is not null) for a persona
// Group by platform, returning the latest version for each platform
// Used by the persona detail page to show which platforms have overrides
```

**Update existing `saveVersion` method** to accept an optional `platform` parameter:

```typescript
async saveVersion(
  personaId: string,
  extractedProfile: unknown,
  sampleStats: { sampleCount: number; samplePlatforms: string[]; totalWordCount: number },
  platform?: string  // NEW — if provided, this is a platform-specific override
): Promise<VoiceProfileVersion>
```

When platform is provided:
- Version numbering is per-platform (not global) — get max version where personaId AND platform match
- Don't update persona.activeVoiceVersionId (that's for the default voice only)

**Update existing voice extraction endpoint** (`src/routes/(app)/personas/[id]/voice/+server.ts`) to accept an optional `platform` query param or body field. When provided:
- Filter writing samples by that platform before extracting
- Pass platform to saveVersion so it creates a platform-specific override
- Include in the AI prompt: "This voice profile is specifically for {platform} writing style"
