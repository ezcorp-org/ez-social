**Create `src/routes/(app)/personas/[id]/calibrate/+page.svelte`** — the calibration flow:

The page guides the user through a 3-step calibration process:

**Step 1: Generate Samples**
- Show persona name and a brief explanation: "Let's calibrate your voice. We'll generate sample replies and you rate how well they sound like you."
- Predefined topic suggestions (3 common social media scenarios — e.g., "someone sharing a hot take on your industry", "a post asking for tool recommendations", "a personal milestone announcement"). User can also type custom topics.
- "Generate Samples" button that POST fetches `/personas/${personaId}/calibrate` with the topics
- Stream the response (same ReadableStream + TextDecoder pattern from voice extraction). Parse the JSON array progressively — show each sample as it arrives.
- State: `step` ($state: 'generate' | 'rate' | 'refining' | 'done'), `samples` ($state array), `ratings` ($state Map)

**Step 2: Rate Samples**
- Show each sample reply with the topic context above it
- Two buttons per sample: "Sounds like me" (thumbs up / green) and "Doesn't sound like me" (thumbs down / red)
- Track ratings in the Map keyed by index
- Once all 3 are rated, show a "Refine Voice" button

**Step 3: Refine Voice**
- On "Refine Voice" click:
  1. POST the ratings to the voice service: save calibration ratings via a form action or API call
  2. Trigger voice re-extraction by POSTing to `/personas/${personaId}/voice` with a special `recalibrate: true` flag (the voice endpoint already has all samples; the feedback is now saved and will be included in the prompt)
  3. Show a streaming progress indicator (same pattern as voice extraction — "Refining your voice profile...")
  4. On completion, show "Voice profile updated! Version N" and a link back to persona detail page

**Step 4: Done**
- Show a success message with before/after comparison hint
- "Back to Persona" link
