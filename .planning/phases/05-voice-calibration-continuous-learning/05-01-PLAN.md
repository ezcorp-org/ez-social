**Add draftFeedback table to schema** following existing patterns (uuid PK, timestamp with timezone, index helper):

- `id`: uuid, defaultRandom, primaryKey
- `postId`: uuid, notNull, references postQueue.id with onDelete 'cascade'
- `messageId`: uuid, notNull, references chatMessages.id with onDelete 'cascade'
- `personaId`: uuid, nullable, references personas.id with onDelete 'set null' (persona at time of feedback, for querying per-persona)
- `action`: varchar(20), notNull — values: 'accepted' | 'rejected' | 'edited'
- `draftText`: text, notNull — the draft text at the time of feedback
- `editedText`: text, nullable — only set when action is 'edited'
- `createdAt`: timestamp with timezone, defaultNow, notNull
- Indexes: on personaId, on postId

- draftFeedback belongs to postQueue (many-to-one)
- draftFeedback belongs to chatMessages (many-to-one)
